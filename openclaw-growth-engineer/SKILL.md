# OpenClaw Growth Engineer

## Use This Skill When

- you want weekly or on-demand product optimization issues created automatically
- you have analytics signals and want concrete file-level implementation tasks
- you want one pipeline from insights to actionable engineering backlog

## Hackathon Scope

- Input: analytics summary JSON (required), RevenueCat summary JSON (optional), Sentry summary JSON (optional)
- Repo scan: keyword-based mapping of signals to likely files/modules
- Output: 3-5 high-priority issue drafts with implementation steps and PR prompt
- Optional: direct GitHub issue creation via API

## Core Rules

- Keep scope bounded: one repo, one data window, max 3-5 issues per run.
- Every generated issue must include:
  - `Problem`
  - `Evidence`
  - `Affected Files / Modules`
  - `Proposed Implementation`
  - `Expected Impact`
  - `Confidence`
  - optional PR prompt
- Prioritize by `priority` first, then by metric delta magnitude.
- Avoid generic “improve UX” tasks without file/module hypotheses.
- If data is weak, still generate issue draft but mark confidence `Low`.

## Setup

1. Run wizard to create non-secret config:
   - `node scripts/openclaw-growth-wizard.mjs`
2. Prepare input files (JSON):
   - `analytics_summary.json` (required)
   - `revenuecat_summary.json` (optional)
   - `sentry_summary.json` (optional)
   - `feedback_summary.json` (optional)
3. Set GitHub token only if you want auto-create:
   - `GITHUB_TOKEN=<token with issue write permission>`
4. Run the script from repo root.

## Commands

Generate drafts only:

```bash
node scripts/openclaw-growth-engineer.mjs \
  --analytics data/openclaw-growth-engineer/analytics_summary.example.json \
  --revenuecat data/openclaw-growth-engineer/revenuecat_summary.example.json \
  --sentry data/openclaw-growth-engineer/sentry_summary.example.json \
  --repo-root . \
  --max-issues 4
```

Wizard + scheduled runner:

```bash
node scripts/openclaw-growth-wizard.mjs

# optional: run feedback ingest API (for domain/webhook input)
node scripts/openclaw-feedback-api.mjs --port 4310

# run once
node scripts/openclaw-growth-runner.mjs \
  --config data/openclaw-growth-engineer/config.json

# run continuously based on configured interval
node scripts/openclaw-growth-runner.mjs \
  --config data/openclaw-growth-engineer/config.json \
  --loop
```

Generate drafts and create GitHub issues:

```bash
GITHUB_TOKEN=ghp_xxx node scripts/openclaw-growth-engineer.mjs \
  --analytics data/openclaw-growth-engineer/analytics_summary.example.json \
  --revenuecat data/openclaw-growth-engineer/revenuecat_summary.example.json \
  --sentry data/openclaw-growth-engineer/sentry_summary.example.json \
  --repo-root . \
  --create-issues \
  --repo owner/repo \
  --labels ai-growth,autogenerated,product
```

## OpenClaw Secrets Mapping

- Store secret in OpenClaw and map/inject to environment variable `GITHUB_TOKEN`.
- Keep read-only keys for data sources separated from write keys.
- Never pass tokens as command arguments.
- If feedback API is exposed publicly, set `FEEDBACK_API_TOKEN` and require `x-feedback-token`.
