Most founders already have enough production data to make better product decisions. The problem is that analytics, subscription metrics, crashes, store reviews, feedback, and code context live in separate places, so the AI coding agent doing the work cannot see the full picture.

AnalyticsCLI and the AI Growth Engineer connect those signals so an agent can propose focused product work, create issue or PR tasks, and help optimize a SaaS or mobile app from real production evidence.

## The Problem

Explain the manual workflow founders use today.

Example:

- check analytics dashboard
- glance at RevenueCat
- ignore crash context until it is urgent
- read a few reviews
- paste screenshots into Codex or Claude Code
- ask for product ideas without giving the agent enough context

## Why This Breaks

Show the tradeoff. Manual analysis is partial, slow, and often disconnected from implementation.

## The Better Loop

Describe the loop:

1. Track explicit production events.
2. Connect revenue, crashes, feedback, store, and code context.
3. Let the AI Growth Engineer rank opportunities.
4. Review the generated issue, PR task, code plan, or marketing advice.
5. Ship with your existing coding-agent subscription.
6. Verify the impact in release analytics.

## FAQ

### Can this use my existing Codex or Claude Code subscription?

Yes. AnalyticsCLI is designed to give your existing coding-agent workflow better product context rather than replace it.

### Does AnalyticsCLI automatically record user behavior?

No. AnalyticsCLI ingests explicit events your app sends.

### Can it create issues or PR tasks?

Yes, when configured. Human review is still recommended before shipping changes.
